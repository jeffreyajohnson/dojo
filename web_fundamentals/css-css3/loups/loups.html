My essays

My last posts depend on each other, and should be read in sequence. Articles not in bold are optional.

  • Assignment Statement Considered Harmful. Pervasive mutable state programs more difficult to work with. While this is old news, most programmers don't know or don't care. It's a pity, because simple remedies exist.
  • Defining Syntatic Sugar. I sometimes hear arguments about what is or is not syntatic sugar. This is an attempt at defining it.
  • Class-based Programming as Syntatic Sugar. Most of the "Object Oriented" actually mean classes. Classes are nothing more than syntatic sugar, at least in statically typed languages such as Java and C++. Sugar is not useless, but seeing past it helps us understand classes better.
  • Taboo "OO". The term "OO: is so overloaded that we should stop using it. Better subsitutes are "classes" and "prototypes".
  • How Class based Programming sucks. Classes are vastely sub-optimal. Functional programming is better.
